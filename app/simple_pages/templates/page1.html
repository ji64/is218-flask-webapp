Git

Branches

Branches allow others to be able to edit files from the master branch and be able to push them to their branch without affecting the master branch. Once ready a user can create a pull request and eventually merge their branch to the master branch.

A new branch can be created with git checkout -b [new branch] or git switch -c [new branch]

To switch between branches use git checkout [branch name] or git switch [branch name]

Merge

Merging allows a user to combine two branches together. This is typically done when a new feature or hotfix has been developed and tested and now needs to be pushed to the master branch. Merging without testing can be dangerous as if someone's code that was not tested gives an error now everyone who has updated to the master branch needs to revert their code back, which is not as easy as it sounds.

To merge two branches use git merge [branch name] or use the GitHub webpage to create a pull request to start a merge

Commit

Committing allows users to upload all of their file changes in order to keep track of what lines of text or code are added or removed and which files were created or deleted. Committing is like taking a snapshot of where the project is at that exact moment in time.

In order to commit you must add the files to be staged by git. Use git add . to add all files in the current directory. Also git add * can be used to add only files that have been changed in the current directory. git add [filename] to only add a specific file.

Once the files have been staged use git commit -m "[message]"

After committing the files use git push origin [branch] to push the change to GitHub which will allow others to see the changes

Repositories

A repository is the location where the code is stored. The user needs to push committed changes in order for the code base to be updated. The repository is also responsible for knowing what branches have been committed and how to merge those branches when requested.

To create a repository first in the directory which the project will be stored in do git init which will initialize a new repository.

Next to link it to GitHub first on the webpage login and create a new repository. Once created a link inorder to push to the repository can be added by doing git remote add origin https://github.com/username/new_repo

After adding the remote link to the repository it is recommended to do your first commit to upload all the initial files. Use git add . to add all files in the directory then commit it with git commit -m "initial commit" then push it with git push origin master
