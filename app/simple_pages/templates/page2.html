Docker

Containers

Containers are lightweight applications that can be run inside of docker which can be installed on almost any computer. The goal of a container is to be able to run exactly the same way on any machine. In order to do this a container is an abstraction at the application layer. This allows containers to share the same host os which for spinning containers up and down can be almost instantaneous compared to a virtual machine.

Images

Images is the portable blueprint that when called will spin up a container with that exact configuration of software. The Image contains all the necessary files in order to run the container so once the image is downloaded it won't have to download or fetch other files.

Commands

Docker

docker ps - shows the running docker containers

docker run [options] [image] - creates a container from the image

docker start [container] - starts an existing container

docker stop [container] - stops an existing container

docker rm [container] - removes a container

docker kill [container] - forcefully stops a container

docker exec [options] [container] [command] - execute a command on the container

Docker Compose

docker-compose up - executes the docker-compose.yml in the current working directory

docker-compose down - stops the containers associated with the docker-compose.yml file

Starting Docker via CLI

To start the container first in the terminal navigate to the project folder where the docker-compose.yml file is located.

Next use docker-compose up --build -d to build and start the container

To get into the container and access its CLI use docker ps to get the containers ID. Then use docker exec -it [container ID] /bin/bash to get access to the terminal.
