OOP Glossary

Terms

instantiation - To create an instance of an object.

object - An abstract data type that holds data and has behavior.

class - A template for creating objects with specific data types and behaviors.

namespace - provides a region which class, methods, and variables can be called from.

constructor - used to initialize an object.

fixture - sample data typically used to fill up databases for testing.

type hint - used to indicate what type of data is required to be passed.

type cast - used to change one data type into another data type.

unit test - used to test code to make sure that any changes to the code does not break the way the code is meant to function.

static - its lifetime is the entire length of the programs runtime.

instance method - methods that belong to an object.

instance property - variables that belong to an object

static method - methods that belong to a class

static property - variables that belong to a class

encapsulation - classes and methods that prevent direct access to variables.

inheritance - allows classes to inherit methods and variables from parent classes.

polymorphism - ability to use multiple different types of object through one interface

abstraction - used to hide unnecessary data that the user's don't need to see

Overview of SOLID - principles in which code following OOP should be designed

Overview of Design Patterns - templates used in order to solve problems.
