Arrange-Act-Assert Testing

What is Arrange-Act-Assert?

Arrange-Act-Assert (AAA) is a format for laying out tests. Using this testing format all tests are focused around a single action. This testing design ensure that every action is tested individually making debugging easier if a problem occurs

How to use AAA testing

This test for testing the creation of the multiplication instance shows the principles of AAA.

def test_calculation_multiplication_instance():
    """Testing the Calculator Multiplication"""
    tuple_list = (1, 2)
    calculation = Multiplication.create(tuple_list)
    assert isinstance(calculation, Multiplication)

The first line, def test_calculation_multiplication_instance(): is the definition of the test. This creates the test and allows the test to be called.

The second line, """Testing the Calculator Multiplication""" is the docstring for the test which is used to describe what the test is ment to accomplish.

The third line, tuple_list = (1, 2) is the arrangement. This is where the necessary setup for the test is done.

The fourth line, calculation = Multiplication.create(tuple_list) is the action. This is the action that is being tested.

The fifth line, assert isinstance(calculation, Multiplication) is the assertion. This is where the output of the action is checked to assert whether the code return the correct output.
